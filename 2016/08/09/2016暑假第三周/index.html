XML



XML（eXtensilbe Markup Language) 指可扩展标记语言
是独立于软件和硬件的信息传输工具

XML简介



XML被设计用来传输和储存数据，其焦点是数据的内容
HTML被设计用来显示数据，其焦点是数据的外观
XML是对HTML的补充

  • XML是一种很像HTML的标记语言
  • XML的设计宗旨是传输数据而不是显示数据
  • XML标签没有被预定义，你需自行定义标签
  • XML被设计为具有自我描述性
  • XML是W3C的推荐标准

XML用途



应用于Web开发的很多方面，常用于简化数据的储存和共享

XML把数据从HTML分离


假如你要在HTML文档中显示动态数据，那么每次数据改变你需要话费大量时间来编辑HTMl。
XML可以使你的数据储存在独立的XML文件里，这样你就可以专注于用HTML和CSS进行显示和布局，并确保修改底层数据不再需要对HTML进行任何的改变。
在通过几行JavaScript代码，你就可以读取一个外部的XML文件，并更新你的网页数据内容。

XML的一系列简化


数据共享：实际上计算机系统和数据使用不兼容的格式来储存数据
XML数据以纯文本格式进行存储，因此提供了一种独立于软件和硬件的数据存储方法。
这可以使得创建不同应用程序可以共享数据变得更加简单。


数据传输：对开发人员来说，其中一项最费时的挑战一直是在互联网上的不兼容系统之间交换数据。
由于可以通过各种不兼容的应用程序来读取数据，以 XML 交换数据降低了这种复杂性。

平台变更：升级到新的系统（硬件或软件平台），总是非常费时的。必须转换大量的数据，不兼容的数据经常会丢失。
XML 数据以文本格式存储。这使得 XML 在不损失数据的情况下，更容易扩展或升级到新的操作系统、新的应用程序或新的浏览器。


数据可用范围提升：不同的应用程序都能够访问您的数据，不仅仅在 HTML 页中，也可以从 XML 数据源中进行访问。
通过 XML，您的数据可供各种阅读设备使用（掌上计算机、语音设备、新闻阅读器等），还可以供盲人或其他残障人士使用。

XML语法


  • 所有的XML元素都必须有一个关闭标签
  • 标签区分大小写
  • 标签必须正确嵌套
  • XML文档必须有根元素（例如HTML里的）
  • 属性值必须加引号
  • 有一些字符具有特殊的意义，要用实体引用来代替这些字符

  • 注释：
  • 以LF存储换行

    ##节点层次

图中文档节点只有一个子节点，就是元素，也叫作文档元素。
文档中其他的元素都在文档元素中，每个文档只有一个文档元素。

Node类型



Node 接口由DOM中所有节点类型实现。
在JavaScript中作为Node类型实现，除了IE，在其他浏览器都可以访问到这个类型。

每个节点有一个nodeType属性，表明节点的类型，一共有12个数值常量来表明这些类型。

nodeName和nodeValue属性

nodeName中保存的是元素的标签名，nodeValue的值始终为null

节点关系

文档中所有的节点之间都存在这样或者呢样的关系。
例如子元素父元素
1.每个节点都有一个childNodes属性，其中有一个NodeList对象（类数组对象，不是Array的实例，可以通过DOM结构的变化来自动反应）。
2.每个节点都有一个parentNode属性，该属性指向文档树中的父节点。
通过childNodes列表中每个节点的previousSibling和nextSibling属性可以访问同一列表中的其他节点。
3.hasChildNodes()方法：该方法在节点包含一或多子节点的情况下返回true。

操作节点（使用下列方法必须先取得父节点）

1.appendChild()
用于在childNodes列表的末尾添加一个节点。
返回新增的节点。
假如传入到appenChild()中的节点已经是文档的一部分了，那结果就是将该节点从原来的位置转移到新位置。（可以将DOM树看成是由一系列指针连接起来的）
2.insertBefore()
接受2个参数：要插入的节点和作为参照的节点。
插入后，被插入的节点会变成参照节点的前一个同胞节点，同时返回被插入节点。
3.replaceChild()
接受2个参数：要插入的节点和要替换的节点。
4.removeChild()
接受一个参数：要移除的节点。
5.cloneNode()
用来创建调用这个方法的节点的一个完全相同的副本
接受一个一个布尔参数，true表示复制节点及其整个子节点树，false表示只复制节点本身。 6.normalize()
处理文档树中的文本节点
（略）

Document类型



JavaScript通过Document类型表示文档。可以表示HTMl页面或者其他基于XML的文档。最常见的是作为HTMLDocument实例的document对象。

文档的子节点

可以是DocumentType，Element，ProcessingInstruction或者Comment。
还有2个内置的访问其节点的快捷方式：documentElement属性（更快捷）和通过childNodes列表访问文档元素。

  • documentElement，firstChild和childNodes[0]的值相同，都指向元素。
  • document.body直接指向元素。
  • 用document.doctype访问<!DOCTYPE>标签中的信息。

    文档信息



    document对象的属性
    1.title属性
    var x=document.title（获取文档标题并可以修改）
    2.URL属性（不可设置）
    包含页面完整的URL
    3.domain属性（可设置，有一系列的规则并且有助于子域框架和内嵌框架的沟通）
    包含页面的域名
    4.referrer属性（不可设置）
    保存链接到当前页面的那个页面的URL（没有源页面的情况下，里面可能包含的是空字符串）

    2.getElementsTagName()
    3.getElementsByName()（name值，返回数组）

    4.document.imges:包含文档中所有的< img >元素。
    5.document.links:包含文档中所有的带href特性的< a >元素。

小结DOM

DOM是语言中立的API，用于访问和操作HTML和XML文档。
DOM由各种节点构成

  • 最基本的就是Node，抽象的表示文档中一个独立的部分，其他类型都继承自Node
  • Document类型表示整个文档，使用document对象，有很多种方式可以查询和获取节点
  • Element节点表示文档中所有的HTML或XML元素，可以用来操作这些元素的内容和特性
  • 其他一些节点类型用来表示文本内容，注释，文档类型，CDATA区域和文档片段
